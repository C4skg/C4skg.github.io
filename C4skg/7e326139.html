网页渲染md美化指南

发表于2023-02-02|更新于2023-03-10|指南
|阅读量:

需求和前提

  1. 在开发独立的可视化工具时，遇到的一个问题，本地写好的 markdown 文档渲染到浏览器页面完全没有美感可言。且美化过程费事费力还费神，因此通过这篇文章来记录和总结网页端渲染 markdown 的美化问题。

  2. 本篇文章只总结了具体的美化和中途可能遇到的一些问题的总结，markdown 转 html 的过程不重复。

  3. 美化过程中所用的第三方库如下

    • highlight.js

      用于实现代码块高亮

    • highlightjs-line-numbers.js

      用于实现代码块左侧行数的显示

    • KaTeX.js

      用于渲染数学公式

代码块显示行号及优化

  1. 下载并使用 highlightjs-line-numbers.js 进行代码块左侧行号的显示

    下载完成后引入js 文件即可，这里采用纯静态引用，因此引用目录中的 dist 目录下的文件即可

  2. 代码高亮颜色影响修复

    直接开启行号显示后，默认的行号显示不够美观，且会使得代码块中的某些文字无法高亮，甚至看不清，这是因为行号显示插件在 pre>code 标签中又加了一层 table 和 tbody 标签来辅助显示行号，更利于排版，但是 table 标签的默认 css 中设置了颜色，覆盖了代码高亮的设置，因此可以通过手动添加 css 方法来避免问题。

    通过将所有被其影响的属性再次进行覆盖，让他们继承他们的父级标签的属性值即可。
